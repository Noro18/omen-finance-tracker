# REQUIREMENT ENGINEERING - Personal Finance Tracker

## Elecitation (rekolla  nesecidade)

**User**
- User (system nia nain)


**Problema**

- User difisil hodi hakerek income no expense
- User la hatene budget ne'e restu hira
- User la hatene osan tama no sai hira iha kada fulan
- User la hatene osan hira mak nia rai ona
- User mos labele hatene relatorio finansa
- User mos labele hatene oinsa mak kondisaun finansa sei akontese iha futuru

**Necesidade User**

- Anota income no expense
- Halo Budget
- Haree relatorio finansa
- Haree current mount
- 
- Haree Dashboard financa
- Bele ajuda predisaun kondisaun financa iha futuru

## Analysis (analiza nesesidade)

iha parte ida ne'e ita sei komprende oinsa sisteman ne halo servisu

**Exemplo Analisa:**

1. User tenke bele:
- aumenta transaksaun
- hamoos transaksaun
- haree lista transaksaun
- halo budget kada kategori
- haree restu budget
- haree osan sira ne'ebe sai ona kada fulan
    ```sei iha button fulan no karik user hili kada fulan sira sei fo sai sira nia dashboard```
- haree predisaun iha futuru


2. sistema tenke bele:
- sura total income
- sura total expense
- sura current mount
- sura restu budget
- halo predisaun uza machine learning


## Spesification (hakerek requeremento sira)
iha step ida ne'e requirement sira sei hakerek clearly.

**Functional Requirement**

feature ne'ebe tenke iha sistema nia laran mak:
- User bele aumenta transaksaun income (husi form insert ba table)
- User bele aumenta transaksaun expense (husi form insert ba table)
- User bele haree lista transaksaun 
- User bele halo budget (sei iha form input ba table)
- Sistema sura restu budget
- Sistema hamosu relatorio financa
- Sistema sura saving
- sistema fo sai dashboard financa
- Sistema bele fo sai predisaun financa uza machine learning

**Non-functional Requirementd**
- Sistema tenke fasil hodi utiliza
- Nia UI ho UX tenke diak no familiar atu nune'e user labele bingun
- Sistema tenke rai dados ho seguru
- Sistema tentke fo sai ho lalais
- Sistema tenke fo sai vizualizasaun data ne'ebe clearly.



```iha parte Validation tuir mai ne'e Requirement nia servisu sei optional kuando imi hakarak lee hotu```
